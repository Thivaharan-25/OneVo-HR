# Internationalization (i18n)

## Strategy

ONEVO launches English-first with **i18n-ready architecture** — all user-facing strings are externalized from day one so that adding languages later is a translation task, not a refactoring task.

## Stack

| Concern | Library | Rationale |
|:--------|:--------|:----------|
| Translation runtime | `next-intl` | Deep Next.js App Router integration, RSC support |
| Message format | ICU MessageFormat | Plurals, genders, selects, numbers, dates |
| Translation storage | JSON files per locale | `/messages/{locale}.json` |
| Date/time formatting | `date-fns` with locale | Locale-aware date formatting |
| Number formatting | `Intl.NumberFormat` | Browser-native, locale-aware |
| Currency formatting | `Intl.NumberFormat` | Currency code from tenant config |

## Usage in Components

### Server Components
```tsx
import { useTranslations } from 'next-intl';

export default function EmployeesPage() {
  const t = useTranslations('employees');

  return (
    <PageHeader title={t('title')} />
  );
}
```

### Client Components
```tsx
'use client';
import { useTranslations } from 'next-intl';

export function EmployeeList({ count }: { count: number }) {
  const t = useTranslations('employees');

  return (
    <p className="text-sm text-muted-foreground">
      {t('employeeCount', { count })}
    </p>
  );
}
```

### Date Formatting
```tsx
import { useFormatter } from 'next-intl';

function DateDisplay({ date }: { date: string }) {
  const format = useFormatter();

  return (
    <time dateTime={date}>
      {format.dateTime(new Date(date), { year: 'numeric', month: 'short', day: 'numeric' })}
    </time>
  );
}
```

### Currency Formatting
```tsx
function CurrencyDisplay({ amount, currency }: { amount: number; currency: string }) {
  const format = useFormatter();

  return (
    <span className="font-mono tabular-nums">
      {format.number(amount, { style: 'currency', currency })}
    </span>
  );
}
```

## Locale Resolution

Priority order:
1. User preference (saved in profile settings)
2. Tenant default locale (from tenant config)
3. Browser `Accept-Language` header
4. Fallback: `en`

```tsx
// middleware.ts
export function middleware(request: NextRequest) {
  const locale = getUserLocale(request) ?? getTenantLocale(request) ?? detectBrowserLocale(request) ?? 'en';
  // ... set locale in headers/cookies
}
```

## Rules for Developers

1. **Never hardcode user-facing strings** — always use `t('key')`
2. **Use ICU MessageFormat for plurals** — don't do `count === 1 ? 'item' : 'items'`
3. **Don't concatenate translated strings** — `t('hello') + ' ' + name` breaks in many languages
4. **Format dates and numbers with locale APIs** — never `date.toLocaleDateString()`
5. **Keep keys descriptive** — `employees.addEmployee` not `btn1`
6. **English JSON is the source of truth** — other locales reference it
