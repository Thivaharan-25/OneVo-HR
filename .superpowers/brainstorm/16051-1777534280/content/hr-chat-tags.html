A — Pinned tag strip at top of chat

HR action tags always pinned above messages. One click fires the action — shows a confirmation card in chat. Tags update based on current state (clocked in → shows Break/Clock Out only).

B — Tag chips in the message composer

HR tags live below the message input, like shortcuts. Developer types or clicks a tag chip → it inserts into the message → sends → backend executes. Works exactly like typing @leave:request manually.
