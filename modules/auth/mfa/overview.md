# Multi-Factor Authentication

**Module:** Auth & Security
**Feature:** MFA

---

## Purpose

Optional MFA support for enhanced security. MFA uses authenticator-app TOTP as the primary method. Email OTP is fallback/recovery only when policy permits it.

## API Endpoints

| Method | Route | Permission | Description |
|:-------|:------|:-----------|:------------|
| POST | `/api/v1/auth/mfa/enable` | Authenticated | Begin TOTP MFA setup |
| POST | `/api/v1/auth/mfa/confirm` | Authenticated | Confirm TOTP setup with authenticator code |
| POST | `/api/v1/auth/mfa/send` | `mfa_pending` | Send/resend email OTP fallback when allowed |
| POST | `/api/v1/auth/mfa/verify` | `mfa_pending` | Verify TOTP code or allowed fallback code |

## Rules

- TOTP codes are 6 digits and generated by the user's authenticator app.
- TOTP secrets are encrypted at rest and are never returned after setup.
- Verification allows only a small clock-skew window.
- Reuse of the same TOTP code inside the accepted window is rejected.
- Email OTP is fallback/recovery only. Fallback OTPs are sent to the user's verified email through the notification/email boundary.
- Raw fallback OTP is never stored; only a temporary hash is stored for verification.
- Fallback OTP expires after 5 minutes, is single-use, and is rate-limited to prevent mailbox flooding.
