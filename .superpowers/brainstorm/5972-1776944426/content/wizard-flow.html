Import Employees — Wizard Flow

Both paths (CSV and PeopleHR) converge at the same validation + confirmation screen

Employee List Screen → "Import Employees" button
↓
Step 1

Choose Import Method

Two cards: "Upload CSV / Excel file"  |  "Connect PeopleHR"

CSV / Excel Path
Step 2a

Upload file (drag & drop). System reads columns, picks Row 1 as sample employee.

Step 3a — Map Fields

Left: actual data from sample employee. Right: ONEVO fields. Admin connects them by dragging/clicking. No abstract schema — real names, real values.

Step 4a — Preview One

"Based on your mapping, this is how John Smith will be imported →" Admin confirms it looks right.

PeopleHR API Path
Step 2b

Enter API key (or select saved connection from Integrations screen). System fetches employee list.

Step 3b — Auto-mapped Preview

System shows first employee, auto-mapped. Admin reviews — "Does this look right?" Confirm or adjust.

Step 4b — Save Connection?

Option: "Save PeopleHR connection to Integrations" — so it's visible on the Integrations screen later.

↓ Both paths converge ↓
Step 5 — Validation

Apply Mapping to All Rows + Show Results

"247 employees ready  ✓  |  3 rows have issues →"   Admin can skip errors or fix them inline. Hierarchy (reporting lines) validated here too — detects circular references, missing managers.

Step 6 — Confirm & Import

Summary screen → "Start Import" button

Shows: total employees, departments detected, managers mapped, errors skipped. Progress bar runs async.

Step 7 — Done

Import Complete

"244 employees imported successfully. 3 skipped (view report)." → Returns to Employee list, now populated.

Integrations Screen Alignment

PeopleHR connections saved during import appear in Settings → Integrations with status (Connected / Last synced), API key management, and import history log. Future connectors (BambooHR, Workday) drop into the same Integrations card pattern.
