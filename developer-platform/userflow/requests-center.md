# Requests Center User Flow

## Purpose

Super Admin reviews demo/trial requests that require platform-side approval. Paid activation is customer self-service and does not enter Requests Center.

## Demo Requests

1. Open Requests Center -> Demo Requests.
2. Review requester, company/tenant details, requested demo access, status, and requested date.
3. Open request details.
4. Validate requested demo access against active Demo Profiles.
5. Approve or reject.
6. Approval creates or updates the demo tenant, notifies the requester, and writes audit history.

## Trial Extension Requests

1. Open Requests Center -> Trial Extension Requests.
2. Review summary stats and request table.
3. Open request details.
4. Review current trial dates, requested days, reason, and usage.
5. Approve with approved extension days or reject.
6. Decision updates tenant state where applicable, notifies the tenant, and writes audit history.
