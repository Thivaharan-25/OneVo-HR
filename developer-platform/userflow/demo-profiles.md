# Demo Profiles User Flow

## Purpose

Super Admin configures demo/trial behavior and upgrade choices for demo tenants.

## Flow

1. Open Demo Profiles from Platform Management.
2. View active and inactive demo profiles.
3. Create or edit a demo profile.
4. Set profile name, trial duration, auto-expire, active status, max employees, demo storage limit, and demo AI token limit.
5. Configure module access level for each demo module: Full Access, View Only, or Archive.
6. Configure feature permissions for each module.
7. Choose allowed subscription plans for upgrade.
8. Choose allowed optional add-ons from those plans.
9. Hide add-ons that are not approved or already included as base modules.
10. Set add-on visibility to Enabled or Show Only.
11. Activate the profile.

## End State

New demo tenants created from the profile receive the configured trial window, shared resource limits, module access, feature permissions, and upgrade choices.
