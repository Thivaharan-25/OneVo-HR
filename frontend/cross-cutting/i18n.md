# Internationalization (i18n)

## Strategy

ONEVO launches English-first with **i18n-ready architecture** — all user-facing strings are externalized from day one so that adding languages later is a translation task, not a refactoring task.

## Stack

| Concern | Library | Rationale |
|:--------|:--------|:----------|
| Translation runtime | `i18next` + `react-i18next` | Works with Vite SPA, mature, wide ecosystem |
| Message format | ICU MessageFormat (`i18next-icu`) | Plurals, genders, selects, numbers, dates |
| Translation storage | JSON files per locale | `public/locales/{locale}/` |
| Date/time formatting | `date-fns` with locale | Locale-aware date formatting |
| Number formatting | `Intl.NumberFormat` | Browser-native, locale-aware |
| Currency formatting | `Intl.NumberFormat` | Currency code from tenant config |

> **Do NOT use `next-intl`** — that library requires Next.js App Router. This project uses `i18next` + `react-i18next` which works with any React setup including Vite.

## Setup

```typescript
// lib/i18n.ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import HttpBackend from 'i18next-http-backend';
import LanguageDetector from 'i18next-browser-languagedetector';

i18n
  .use(HttpBackend)
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    fallbackLng: 'en',
    supportedLngs: ['en', 'ta', 'si'],
    ns: ['common', 'employees', 'leave', 'workforce', 'auth', 'org', 'settings'],
    defaultNS: 'common',
    backend: {
      loadPath: '/locales/{{lng}}/{{ns}}.json',
    },
    detection: {
      order: ['localStorage', 'navigator'],
      caches: ['localStorage'],
    },
    interpolation: {
      escapeValue: false, // React already escapes
    },
  });

export default i18n;
```

```tsx
// src/main.tsx — import before mounting
import './lib/i18n';
import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';
import { App } from './App';

createRoot(document.getElementById('root')!).render(
  <StrictMode><App /></StrictMode>
);
```

## Usage in Components

### Standard Usage

```tsx
import { useTranslation } from 'react-i18next';

export function EmployeesPage() {
  const { t } = useTranslation('employees');

  return (
    <PageHeader title={t('title')} />
  );
}
```

### Plurals

```tsx
import { useTranslation } from 'react-i18next';

export function EmployeeCount({ count }: { count: number }) {
  const { t } = useTranslation('employees');

  return (
    <p className="text-sm text-muted-foreground">
      {t('employeeCount', { count })}
    </p>
  );
}
```

### Date Formatting

```tsx
import { format } from 'date-fns';
import { useTranslation } from 'react-i18next';
import { enUS, ta } from 'date-fns/locale';

const LOCALE_MAP: Record<string, Locale> = { en: enUS, ta };

function DateDisplay({ date }: { date: string }) {
  const { i18n } = useTranslation();
  const locale = LOCALE_MAP[i18n.language] ?? enUS;

  return (
    <time dateTime={date}>
      {format(new Date(date), 'dd MMM yyyy', { locale })}
    </time>
  );
}
```

### Currency Formatting

```tsx
function CurrencyDisplay({ amount, currency }: { amount: number; currency: string }) {
  const { i18n } = useTranslation();

  return (
    <span className="font-mono tabular-nums">
      {new Intl.NumberFormat(i18n.language, { style: 'currency', currency }).format(amount)}
    </span>
  );
}
```

## Locale Resolution

Priority order resolved at app startup via i18next `LanguageDetector`:
1. User preference saved in `localStorage`
2. Browser `navigator.language`
3. Fallback: `en`

Tenant default locale (from tenant config) can override by calling `i18n.changeLanguage(tenantLocale)` after the auth session loads.

## Rules for Developers

1. **Never hardcode user-facing strings** — always use `t('key')`
2. **Use i18next plural keys** (`_one`, `_other`, `_zero`) — don't do `count === 1 ? 'item' : 'items'`
3. **Don't concatenate translated strings** — `t('hello') + ' ' + name` breaks in many languages. Use interpolation: `t('greeting', { name })`
4. **Format dates and numbers with locale APIs** — never `date.toLocaleDateString()` without a locale
5. **Keep keys descriptive** — `employees.addEmployee` not `btn1`
6. **English JSON is the source of truth** — other locales reference it
